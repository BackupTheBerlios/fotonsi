Objeto Documento
  • TDocumento: Representa el comportamiento de una página web de forma global; es decir, en este nivel se pueden tratar los eventos de la página. De esta forma, es posible 'simular' atajos de teclado (algunas combinaciones de teclas no se pueden sobreescribir: F3, ALT-F4...etc.), establecer el comportamiento del ratón (desabilitar el botón derecho, asociarlo a una función de usuario, que aparezca un menú...etc.).
      AddAcelerator (tecla, shift, alt, ctrl, acción): Esta función es la encargada de asignar a una combinación de teclas dadas, una función arbitraria. Hay una serie de constantes predefinidas que especifican la tecla shift, alt ó ctrl. Si se pone dicho valor en uno de los parámetros, automáticamente se asocia a la tecla inicial más el modificador/es que se hayan asignados.
      setRoot: Establece el documento actual como 'activo'. Solo puede existir un único componente activo en una página, aunque se pueden definir varios y decidir en que momento se activa uno u otro.
      EnableRightButton(acción) / DisableRightButton: Habilita/Desabilita el botón derecho del ratón. La función de activación, si se le pasa el parámetro, la ejecuta cuando se pulsa el botón derecho del ratón. NOTA: A nivel de documento, solo funciona en Explorer.

    Objeto Formulario
  • TFormulario: Cada formulario existente en la página está representado por un objeto de este tipo. Entre sus cometidos está la comprobación del mismo.
      AddField (campo): Enlaza el campo de formulario actual con el formulario. De esta forma, se puede comprobar el formulario a la hora de enviar la información al servidor.

    Objetos Formulario
  • BaseFormulario: Es la base de toda la jerarquía de objetos de formularios. Aquí se definen los atributos y métodos comunes a todas las instancias de los objetos generados.
      setMessageError () / setNoMessageError(): Cuando se detecta un error, hace
      setNull() / setNoNull(): El valor del objeto no puede o puede estar vacío.
      setFocus() / setNoFocus(): En caso de error, el campo debe coger el foco o no
      getNull() / getFocus() / getMessageError() : Obtener si el objeto es requerido o no / obtener si el foco se coje antes un error o no / obtención del mensaje de error por pantalla o se devuelve un código de error a la función de usuario (gestión propia de errores).
      AddEvent(): Esta es la base para el manejo de los eventos de 'forma transparnte'. La idea en un principio, es que los distintos objetos gestionen sus propios eventos.
    Métodos comunes a todos los objetos:
      valor(): Valor actual del campo.
      comprobar(): Comprueba si el valor actual es válido, en función de las restricciones impuestas a cada objeto del formulario.
      Error(): Muestra el último mensaje de error generado.

  • ControlNumero:
      setMinimo(min) / setMaximo(max): Establece el valor mínimo y máximo del rango válido.
      setRange(min, max): Establece el rango completo válido.
      setNoRange(): Elimina la comprobación de rangos.
      setDecimal() / setNoDecimal(): Se permiten números reales o solo entero.
      getDecimal(): Obtiene si el número se define como número real o solo entero.
